# Contributing / Mitwirken

NeoDex is currently developed as a private project.

## Workflow

1. Create or select an issue.
2. Work in a dedicated branch.
3. Keep game-specific data inside its package.
4. Keep reusable functionality inside `core`.
5. Update tests and documentation.
6. Open a pull request before merging into `main`.

## Quality rules

- Do not silently invent game data.
- Record sources and supported game versions.
- Clearly label uncertain or incomplete information.
- Keep the interface calm and accessible.
- Do not modify the frozen master design without an explicit design decision.
- Add visible user value with every feature.

## Sprache

Dokumentation kann auf Deutsch oder Englisch eingereicht werden. Benutzertexte müssen langfristig in beiden Oberflächensprachen verfügbar sein.
